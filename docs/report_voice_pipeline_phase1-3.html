System Architecture

End-to-end voice interaction: user speaks into a 4-microphone array, the AI processes the request (including FPGA LED control), and responds through a speaker.

4x INMP441 Mic ──I2S──> FPGA PL (I2S Rx x4) ──DMA──> PS DDR | Python Voice Pipeline (PS ARM): VAD (webrtcvad) > STT (Groq Whisper API) > PicoClaw WebSocket > LLM (Claude AI) > Agent Loop (led_ctrl, tools) < Response text > TTS (OpenAI / espeak-ng) | PS DDR ──DMA──> FPGA PL (I2S Tx) ──I2S──> MAX98357A ──> Speaker

Conversation Flow
User says: "Turn on LED 1" > INMP441 mic array → I2S → FPGA PL → DMA → PS ALSA > VAD detects speech (800ms silence = segment complete) > Groq Whisper STT → "Turn on LED 1" > PicoClaw WebSocket → Claude AI agent loop > exec(led_ctrl on 1) → FPGA LED turns on > Response: "LED 1 is now on." > OpenAI TTS → audio PCM > PS ALSA → DMA → FPGA PL I2S Tx → MAX98357A → Speaker User hears: "LED 1 is now on."

Phase 4: Beamforming + Full Integration
